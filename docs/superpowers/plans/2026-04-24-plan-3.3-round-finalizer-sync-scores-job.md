# Plano 3.3 — RoundFinalizerService + SyncScoresJob

> **For agentic workers:** REQUIRED SUB-SKILL: superpowers:executing-plans + superpowers:test-driven-development.

**Goal:** Duas peças que fecham o ciclo da rodada:

1. **`RoundFinalizerService`** — dado um round com match `finished`, calcula `guess.points` (via `calculatePoints`), recalcula `scores` do usuário/season (full recompute) e flipa `round.status → finished`.
2. **`SyncScoresJob`** — roda a cada 10 min, chama `RefreshMatchService` pra todo match ativo não-finished (assim `match.status: scheduled → live → finished` acompanha o provider), e quando match vira `finished` **e** round está `closed`, dispara o finalizer.

**Architecture:**
- Finalizer: orquestrador puro-ish (DI de repos). Idempotente — rodar duas vezes no mesmo round produz o mesmo resultado.
- Job: loop sobre matches ativos não-finished. Erro por match não derruba o job.
- `scores` é recalculado por **full recompute** (soma todos os `guesses.points` do user/season), não incremental — idempotência garantida mesmo se retry duplicado.

**Pré-requisito:** Planos 3.1 e 3.2 concluídos.

**Critério de aceitação:**
- Finalizer: 4+ unit tests (compute correto, upsert scores, idempotência, short-circuit se match não finished).
- Job: funcional cobrindo: match ainda scheduled → só refresh; match vira finished + round closed → finaliza; match vira finished + round não-closed → skip finalize.
- Typecheck + suite passa.

- [ ] **Step 3: Implementar `RoundFinalizerService`**

```ts
import { inject } from '@adonisjs/core'
import { calculatePoints } from '#services/guess_scoring_service'
import GuessRepository from '#repositories/guess_repository'
import MatchRepository from '#repositories/match_repository'
import RoundRepository from '#repositories/round_repository'
import ScoreRepository from '#repositories/score_repository'

@inject()
export default class RoundFinalizerService {
  constructor(
    private roundRepository: RoundRepository,
    private matchRepository: MatchRepository,
    private guessRepository: GuessRepository,
    private scoreRepository: ScoreRepository
  ) {}

  async finalize(roundId: string): Promise<void> {
    const round = await this.roundRepository.findByIdOrFail(roundId)
    const match = await this.matchRepository.findByRoundId(round.id)
    if (!match || match.status !== 'finished') {
      throw new Error('match não finalizado — não é possível finalizar o round')
    }
    if (match.homeScore === null || match.awayScore === null) {
      throw new Error('match finalizado mas sem placar — bug no provider/refresh')
    }

    const final = { finalHome: match.homeScore, finalAway: match.awayScore }

    // 1. Calcula points por guess desse match
    const guesses = await this.guessRepository.listByMatchId(match.id)
    const affectedUserIds = new Set<string>()
    for (const guess of guesses) {
      const points = calculatePoints(
        { guessHome: guess.guessHome, guessAway: guess.guessAway },
        final
      )
      await this.guessRepository.update(guess, { points } as any)
      affectedUserIds.add(guess.userId)
    }

    // 2. Recalcula scores de cada user afetado (full recompute pra idempotência)
    const seasonId = round.seasonId
    for (const userId of affectedUserIds) {
      const userGuesses = await this.guessRepository.listBySeasonAndUser(seasonId, userId)
      const totalPoints = userGuesses.reduce((acc, g) => acc + (g.points ?? 0), 0)
      const exactScoresCount = userGuesses.filter((g) => g.points === 3).length
      await this.scoreRepository.upsert(userId, seasonId, { totalPoints, exactScoresCount })
    }

    // 3. Flipa round
    await this.roundRepository.update(round, { status: 'finished' } as any)
  }
}
```

```ts
import { inject } from '@adonisjs/core'
import logger from '@adonisjs/core/services/logger'
import MatchRepository from '#repositories/match_repository'
import RefreshMatchService from '#services/refresh_match_service'
import RoundFinalizerService from '#services/round_finalizer_service'

export interface SyncScoresRun {
  matchId: string
  roundId: string
  refreshed: boolean
  finalized: boolean
  error?: string
}

export interface SyncScoresReport {
  runs: SyncScoresRun[]
}

@inject()
export default class SyncScoresJob {
  constructor(
    private matchRepository: MatchRepository,
    private refreshMatchService: RefreshMatchService,
    private roundFinalizerService: RoundFinalizerService
  ) {}

  async run(): Promise<SyncScoresReport> {
    const matches = await this.matchRepository.listActiveNonFinished()
    const runs: SyncScoresRun[] = []

    for (const match of matches) {
      try {
        const report = await this.refreshMatchService.refresh(match.id)
        let finalized = false

        // Recarrega pra ver o status pós-refresh
        const fresh = await this.matchRepository.findByIdOrFail(match.id)
        if (fresh.status === 'finished' && match.round.status === 'closed') {
          await this.roundFinalizerService.finalize(match.roundId)
          finalized = true
        }

        runs.push({
          matchId: match.id,
          roundId: match.roundId,
          refreshed: report.updated,
          finalized,
        })
      } catch (err) {
        const msg = err instanceof Error ? err.message : String(err)
        logger.error({ matchId: match.id, err: msg }, 'SyncScoresJob: falha em match')
        runs.push({
          matchId: match.id,
          roundId: match.roundId,
          refreshed: false,
          finalized: false,
          error: msg,
        })
      }
    }

    logger.info({ runs }, 'SyncScoresJob: finished')
    return { runs }
  }
}
```

## Resumo do Plano 3.3

- `RoundFinalizerService` fecha o ciclo: guess.points calculados, scores recalculados (full recompute), round → finished.
- `SyncScoresJob` mantém `match.status` sempre em dia com provider (o ponto que você levantou), e aciona finalizer quando match vira finished + round está closed.
- Dupla proteção: finalizer só roda se match está finished (validação interna), e job só o chama se round está closed (guarda externa).

**Decisões:**
- Full recompute em `scores` ao invés de incremental: idempotência > performance (10 usuários, ~38 rounds = ~380 guesses/season, trivial).
- `listActiveNonFinished` filtra por `season.is_active` — evita refresh de temporadas antigas desativadas.
- Finalizer lança erro se match não finished: guarda defensiva contra chamadas manuais erradas (no job, o check de round.status evita isso).
- Ranking endpoints (Fase 1.8) continuam funcionando sem mudança — leem da `scores` table que agora é populada automaticamente.
