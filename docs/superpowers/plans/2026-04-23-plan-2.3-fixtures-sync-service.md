> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development ou superpowers:executing-plans.

**Goal:** Orquestrar o fluxo completo de sync de uma temporada — buscar fixtures por rodada, buscar standings, escolher o jogo da rodada via picker e criar/preservar a linha em `matches`. Substituir o stub em `SeasonsController.sync` por implementação real.

**Architecture:** Service `FixturesSyncService` em `app/services/fixtures_sync_service.ts` via DI. Recebe `seasonId`, percorre as 38 rodadas, só processa rodadas sem `Match` associado (idempotente). **Tech Stack:** Adonis 7 (DI), Lucid, Japa, axios mockado via `container.swap`.

**Spec:** `docs/superpowers/specs/2026-04-20-palpites-brasileirao-design.md` (seção 4.2).

**Critério de aceitação:**
- `POST /api/v1/seasons/:id/sync` retorna `{ created, skipped, errors }` em vez de stub.
- Rodadas sem match → match criado baseado no picker.
- Rodadas com match existente → skip (não sobrescreve).
- Testes funcionais cobrem happy path + idempotência, usando `container.swap` pra mockar `ApiFootballClient`.

import type { FixtureCandidate, StandingEntry } from '#services/featured_match_picker'
import type {
  ApiFootballFixture,
  ApiFootballStandingsResponse,
} from './types.js'

export function toFixtureCandidate(f: ApiFootballFixture): FixtureCandidate {
  return {
    externalId: f.fixture.id,
    homeTeamId: f.teams.home.id,
    homeTeamName: f.teams.home.name,
    awayTeamId: f.teams.away.id,
    awayTeamName: f.teams.away.name,
    kickoffAt: new Date(f.fixture.date),
  }
}

export function flattenStandings(
  response: ApiFootballStandingsResponse[]
): StandingEntry[] {
  const out: StandingEntry[] = []
  for (const league of response) {
    for (const group of league.league.standings) {
      for (const entry of group) {
        out.push({ teamId: entry.team.id, points: entry.points })
      }
    }
  }
  return out
}
```

import { inject } from '@adonisjs/core'
import { DateTime } from 'luxon'
import ApiFootballClient from '#integrations/api_football/client'
import {
  toFixtureCandidate,
  flattenStandings,
} from '#integrations/api_football/mappers'
import {
  pickFeaturedMatch,
  type StandingEntry,
} from '#services/featured_match_picker'
import MatchRepository from '#repositories/match_repository'
import RoundRepository from '#repositories/round_repository'
import SeasonRepository from '#repositories/season_repository'

export interface SyncReport {
  seasonId: string
  created: number
  skipped: number
  errors: Array<{ roundNumber: number; reason: string }>
}

@inject()
export default class FixturesSyncService {
  constructor(
    private client: ApiFootballClient,
    private seasonRepository: SeasonRepository,
    private roundRepository: RoundRepository,
    private matchRepository: MatchRepository
  ) {}

  async syncSeason(seasonId: string): Promise<SyncReport> {
    const season = await this.seasonRepository.findByIdOrFail(seasonId)
    const report: SyncReport = { seasonId, created: 0, skipped: 0, errors: [] }

    const standings = flattenStandings(
      await this.client.fetchStandings(season.externalLeagueId, season.year)
    )

    const rounds = await this.roundRepository.listBySeason(seasonId)
    for (const round of rounds) {
      const existing = await this.matchRepository.findByRoundId(round.id)
      if (existing) {
        report.skipped++
        continue
      }

      try {
        await this.syncRound(season, round, standings)
        report.created++
      } catch (err) {
        report.errors.push({
          roundNumber: round.number,
          reason: err instanceof Error ? err.message : String(err),
        })
      }
    }

    return report
  }

  private async syncRound(
    season: { year: number; externalLeagueId: number },
    round: { id: string; number: number },
    standings: StandingEntry[]
  ) {
    const roundLabel = `Regular Season - ${round.number}`
    const fixtures = await this.client.fetchFixturesByRound(
      season.externalLeagueId,
      season.year,
      roundLabel
    )
    const candidates = fixtures.map(toFixtureCandidate)
    const pick = pickFeaturedMatch(candidates, standings)
    if (!pick.ok) throw new Error(pick.reason)

    await this.matchRepository.create({
      roundId: round.id,
      externalId: pick.match.externalId,
      homeTeam: pick.match.homeTeamName,
      awayTeam: pick.match.awayTeamName,
      kickoffAt: DateTime.fromJSDate(pick.match.kickoffAt),
      status: 'scheduled',
    })
  }
}
```

    { seasonId: season.id, number: 2 },

  ])

  app.container.swap(ApiFootballClient, () => fake as any)
  try {
    const res = await client.post(`/api/v1/seasons/${season.id}/sync`).headers(HEADERS)
    res.assertStatus(200)
    assert.equal(res.body().created, 2)
    assert.equal(res.body().skipped, 0)
    assert.lengthOf(res.body().errors, 0)
  } finally {
    app.container.restore(ApiFootballClient)
  }
})

test('POST /seasons/:id/sync é idempotente (não sobrescreve)', async ({ client, assert }) => {

    assert.equal(second.body().created, 0)

## Resumo do Plano 2.3

Ao final:
- `POST /api/v1/seasons/:id/sync` faz o fluxo completo.
- Idempotente: rodadas com match não são tocadas.
- Mappers puros (3 unit tests) + orquestrador com DI + 2 testes funcionais com `container.swap`.
- Mock helper reutilizável em `tests/helpers/api_football_mock.ts` (vai ser usado também no Plano 2.4).

**Decisões:**
- Report `{ created, skipped, errors }` — útil pra debug e futuros logs.
- Erros por rodada não interrompem o resto — cada rodada roda isolada.
- `roundLabel = 'Regular Season - N'` é o formato que a API-Football usa pro Brasileirão. Se divergir em prod, ajustar numa iteração.
