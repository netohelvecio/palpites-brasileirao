# Plan 6.2 — GitHub Actions CI Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Pipeline de qualidade rodando automaticamente em todo `push` e `pull_request`: typecheck + lint + prettier --check + 157 testes contra Postgres real.

**Architecture:** Workflow único `.github/workflows/ci.yml` no monorepo. Job `quality` em `ubuntu-latest` com Postgres 16 como service container. `pnpm/action-setup@v4` antes de `actions/setup-node@v4` (ordem correta pra cache pnpm funcionar).

**Tech Stack:** GitHub Actions, pnpm 9, Node 24.14.0, postgres:16-alpine.

**Spec:** `docs/superpowers/specs/2026-04-27-deploy-design.md`

**Depende de:** Plan 6.1 (precisa do script `format:check`).

### Task 1: Criar `.github/workflows/ci.yml`

**Files:**
- Create: `.github/workflows/ci.yml`

- [ ] **Step 1: Criar diretório se não existe**

```bash
mkdir -p .github/workflows
```

- [ ] **Step 2: Conteúdo do workflow**

```yaml
name: ci

on:
  push:
    branches-ignore: [gh-pages]
  pull_request:

concurrency:
  group: ci-${{ github.ref }}
  cancel-in-progress: true

jobs:
  quality:
    runs-on: ubuntu-latest
    services:
      postgres:
        image: postgres:16-alpine
        env:
          POSTGRES_USER: palpites
          POSTGRES_PASSWORD: palpites
          POSTGRES_DB: palpites_test
        ports:
          - 5433:5432
        options: >-
          --health-cmd "pg_isready -U palpites"
          --health-interval 5s
          --health-timeout 3s
          --health-retries 10

    steps:
      - uses: actions/checkout@v4

      - uses: pnpm/action-setup@v4
        with:
          version: 9

      - uses: actions/setup-node@v4
        with:
          node-version: 24.14.0
          cache: pnpm

      - name: Install
        run: pnpm install --frozen-lockfile

      - name: Typecheck (api)
        run: pnpm --filter @palpites/api typecheck

      - name: Lint (api)
        run: pnpm --filter @palpites/api lint

      - name: Prettier (api)
        run: pnpm --filter @palpites/api format:check

      - name: Test (api)
        env:
          DB_HOST: 127.0.0.1
          DB_PORT: 5433
          DB_USER: palpites
          DB_PASSWORD: palpites
          DB_DATABASE: palpites_test
          NODE_ENV: test
        run: pnpm --filter @palpites/api test
```

`concurrency` cancela runs antigos do mesmo branch quando você empurra commit novo — evita fila e gasto à toa.

- [ ] **Step 3: Verificar que o `.env.test` da API casa com as envs do service Postgres**

Run: `cat apps/api/.env.test`
Expected: `DB_HOST=localhost`, `DB_PORT=5433`, `DB_USER=palpites`, `DB_PASSWORD=palpites`, `DB_DATABASE=palpites_test`.

Se o `.env.test` não existir ou divergir, ajustar pra que o teste local e o CI rodem com o mesmo banco. (Pode ser que o `.env.test` use `DB_HOST=localhost` enquanto o CI envia `DB_HOST=127.0.0.1` via env-vars do step — Adonis envs por step sobrescrevem `.env.test`.)

- [ ] **Step 3: Se falhar, debugar**

Cenários comuns:
- **Prettier falha:** rodar `pnpm --filter @palpites/api format` localmente, commitar formatação e empurrar de novo.
- **Test falha por timeout do Postgres:** aumentar `--health-retries` no service.
- **Typecheck falha:** ajustar tipos antes de seguir.
