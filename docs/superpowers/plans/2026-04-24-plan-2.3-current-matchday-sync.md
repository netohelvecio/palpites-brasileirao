# Plano 2.3 — Current Matchday Sync

> **For agentic workers:** REQUIRED SUB-SKILL: superpowers:executing-plans + superpowers:test-driven-development.

**Goal:** Implementar `POST /api/v1/seasons/:id/sync` que busca **apenas a rodada atual** da football-data.org, cria round sob demanda (se não existir) e escolhe o jogo via picker. Idempotente.

**Architecture:**
- Mappers puros em `app/integrations/football_data/mappers.ts` convertem payloads → domínio do picker.
- `FixturesSyncService.syncCurrentMatchday(seasonId)` orquestra:
  1. `GET /competitions/BSA/standings` → extrai `currentMatchday`, year (de `startDate`) e a `table` TOTAL.
  2. Garante `Round(seasonId, matchday)` — cria com `status=pending` se não existir.
  3. Se o round já tem `Match`, skip.
  4. `GET /competitions/BSA/matches?season=Y&matchday=N` → converte em `FixtureCandidate[]`.
  5. `pickFeaturedMatch(candidates, standings)` → cria `Match`.

Diferente do plano 2.3 antigo: **2 requests por sync** (não 38+), e **criação lazy de Round**.

**Tech Stack:** Adonis 7 (DI), Lucid, Japa, `container.swap` pra mockar `FootballDataClient`.

**Pré-requisito:** Planos 2.0, 2.1, 2.2 concluídos.

**Critério de aceitação:**
- `POST /api/v1/seasons/:id/sync` retorna `{ seasonId, currentMatchday, created, skipped, reason? }`.
- Mappers têm 4 unit tests (fixture, standings, currentMatchday, year).
- Testes funcionais cobrem: happy path (cria round + match), idempotência (2ª chamada skip), sem fixtures (reason).

```ts
import type { FixtureCandidate, StandingEntry } from '#services/featured_match_picker'
import type { FootballDataMatch, FootballDataStandingsResponse } from './types.js'

export function toFixtureCandidate(m: FootballDataMatch): FixtureCandidate {
  return {
    externalId: m.id,
    homeTeamId: m.homeTeam.id,
    homeTeamName: m.homeTeam.name,
    awayTeamId: m.awayTeam.id,
    awayTeamName: m.awayTeam.name,
    kickoffAt: new Date(m.utcDate),
  }
}

export function flattenStandings(res: FootballDataStandingsResponse): StandingEntry[] {
  const total = res.standings.find((s) => s.type === 'TOTAL')
  if (!total) return []
  return total.table.map((e) => ({ teamId: e.team.id, points: e.points }))
}

export function extractCurrentMatchday(res: FootballDataStandingsResponse): number {
  return res.season.currentMatchday
}

export function extractSeasonYear(res: FootballDataStandingsResponse): number {
  return new Date(res.season.startDate).getUTCFullYear()
}
```

```ts
import { inject } from '@adonisjs/core'
import { DateTime } from 'luxon'
import FootballDataClient from '#integrations/football_data/client'
import {
  toFixtureCandidate,
  flattenStandings,
  extractCurrentMatchday,
  extractSeasonYear,
} from '#integrations/football_data/mappers'
import { pickFeaturedMatch } from '#services/featured_match_picker'
import MatchRepository from '#repositories/match_repository'
import RoundRepository from '#repositories/round_repository'
import SeasonRepository from '#repositories/season_repository'

export interface SyncReport {
  seasonId: string
  currentMatchday: number
  created: boolean
  skipped: boolean
  reason?: string
}

@inject()
export default class FixturesSyncService {
  constructor(
    private client: FootballDataClient,
    private seasonRepository: SeasonRepository,
    private roundRepository: RoundRepository,
    private matchRepository: MatchRepository
  ) {}

  async syncCurrentMatchday(seasonId: string): Promise<SyncReport> {
    const season = await this.seasonRepository.findByIdOrFail(seasonId)

    const standingsRes = await this.client.fetchStandings(season.externalCompetitionCode)
    const currentMatchday = extractCurrentMatchday(standingsRes)
    const year = extractSeasonYear(standingsRes)

    let round = await this.roundRepository.findBySeasonAndNumber(seasonId, currentMatchday)
    if (!round) {
      round = await this.roundRepository.create({
        seasonId,
        number: currentMatchday,
        status: 'pending',
      } as any)
    }

    const existing = await this.matchRepository.findByRoundId(round.id)
    if (existing) {
      return {
        seasonId,
        currentMatchday,
        created: false,
        skipped: true,
        reason: 'round já possui match',
      }
    }

    const matches = await this.client.fetchMatchesByMatchday(
      season.externalCompetitionCode,
      year,
      currentMatchday
    )
    const candidates = matches.map(toFixtureCandidate)
    const standings = flattenStandings(standingsRes)
    const pick = pickFeaturedMatch(candidates, standings)

    if (!pick.ok) {
      return {
        seasonId,
        currentMatchday,
        created: false,
        skipped: false,
        reason: pick.reason,
      }
    }

    await this.matchRepository.create({
      roundId: round.id,
      externalId: pick.match.externalId,
      homeTeam: pick.match.homeTeamName,
      awayTeam: pick.match.awayTeamName,
      kickoffAt: DateTime.fromJSDate(pick.match.kickoffAt),
      status: 'scheduled',
    } as any)

    return { seasonId, currentMatchday, created: true, skipped: false }
  }
}
```

## Resumo do Plano 2.3

- Sync processa **só o matchday atual** (~2 requests por chamada) → cabe em 10 req/min.
- Round criada sob demanda; match criado via picker.
- 4 unit tests de mapper + 3 funcionais com `container.swap`.
- Mock helper `FakeFootballDataClient` reutilizável no plano 2.4.

**Decisões registradas:**
- Ano da season derivado de `season.startDate` (ano de início do Brasileirão).
- `round.status = 'pending'` na criação — abertura real vem no plano 3 (scheduler).
- Se `matches=[]` retorna `reason` mas status 200 (não é erro do caller — pode simplesmente ainda não ter fixtures cadastrados no provider).
- Se round já existe com match → skip idempotente.
