# Plan 6.1 — Containerização (Dockerfile + compose.prod + Caddy) Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Aplicação completa (API + Postgres + Caddy) rodando localmente em modo produção via `docker compose -f docker-compose.prod.yml up`, com imagem reproduzível ARM64-compatível.

**Architecture:** Dockerfile multi-stage `deps → builder → runtime` produz imagem `node:24-alpine` < 200 MB. `docker-compose.prod.yml` orquestra `caddy` + `api` + `postgres` com 2 redes (`edge`/`app`), volumes nomeados (`postgres_data`, `whatsapp_auth`, `caddy_data`, `caddy_config`) e Docker secret pra senha do banco. Pra testar local sem TLS, criamos um `Caddyfile` adicional que escuta em `:8080` HTTP.

**Tech Stack:** Docker multi-stage, Compose v2, Caddy 2-alpine, postgres:16-alpine, node:24-alpine, pnpm 9, corepack.

**Spec:** `docs/superpowers/specs/2026-04-27-deploy-design.md`

- [ ] **Step 2: Criar Dockerfile com 3 stages**

```dockerfile
# syntax=docker/dockerfile:1.7
ARG NODE_VERSION=24-alpine

FROM node:${NODE_VERSION} AS deps
WORKDIR /repo
RUN corepack enable && corepack prepare pnpm@9 --activate
COPY pnpm-workspace.yaml pnpm-lock.yaml package.json ./
COPY packages/shared/package.json packages/shared/
COPY apps/api/package.json apps/api/
RUN --mount=type=cache,id=pnpm,target=/root/.local/share/pnpm/store \
    pnpm install --frozen-lockfile

FROM node:${NODE_VERSION} AS builder
WORKDIR /repo
RUN corepack enable && corepack prepare pnpm@9 --activate
COPY --from=deps /repo/node_modules ./node_modules
COPY --from=deps /repo/packages/shared/node_modules ./packages/shared/node_modules
COPY --from=deps /repo/apps/api/node_modules ./apps/api/node_modules
COPY pnpm-workspace.yaml pnpm-lock.yaml package.json turbo.json ./
COPY packages packages
COPY apps/api apps/api
RUN pnpm --filter @palpites/api build

FROM node:${NODE_VERSION} AS runtime
ENV NODE_ENV=production
WORKDIR /app
RUN apk add --no-cache curl tini
COPY --from=builder /repo/apps/api/build ./
COPY --from=deps /repo/node_modules ../node_modules
COPY --from=deps /repo/apps/api/node_modules ./node_modules
RUN chown -R node:node /app
USER node
EXPOSE 3333
HEALTHCHECK --interval=30s --timeout=5s --start-period=20s --retries=3 \
  CMD curl -f http://127.0.0.1:3333/health || exit 1
ENTRYPOINT ["/sbin/tini", "--"]
CMD ["node", "bin/server.js"]
```

### Task 4: Criar `docker-compose.prod.yml` na raiz do repo

**Files:**
- Create: `docker-compose.prod.yml`

- [ ] **Step 1: Conteúdo do compose**

```yaml
services:
  caddy:
    image: caddy:2-alpine
    ports:
      - "80:80"
      - "443:443"
    volumes:
      - ./infra/Caddyfile:/etc/caddy/Caddyfile:ro
      - caddy_data:/data
      - caddy_config:/config
    networks: [edge]
    restart: unless-stopped
    depends_on:
      - api

  api:
    image: ${API_IMAGE:-ghcr.io/CHANGE_ME/palpites-api:latest}
    env_file: ./apps/api/.env.production
    volumes:
      - whatsapp_auth:/app/storage/whatsapp-auth
    networks: [edge, app]
    depends_on:
      postgres:
        condition: service_healthy
    restart: unless-stopped

  postgres:
    image: postgres:16-alpine
    environment:
      POSTGRES_USER: palpites
      POSTGRES_PASSWORD_FILE: /run/secrets/db_password
      POSTGRES_DB: palpites_prod
    volumes:
      - postgres_data:/var/lib/postgresql/data
    secrets:
      - db_password
    healthcheck:
      test: ["CMD-SHELL", "pg_isready -U palpites -d palpites_prod"]
      interval: 5s
      timeout: 3s
      retries: 10
    networks: [app]
    restart: unless-stopped

volumes:
  postgres_data:
  whatsapp_auth:
  caddy_data:
  caddy_config:

networks:
  edge:
  app:

secrets:
  db_password:
    file: ./infra/secrets/db_password.txt
```

`API_IMAGE` é resolvido pelo runner: em prod sai do GHCR (`ghcr.io/<user>/palpites-api:latest`), em testes locais o usuário pode passar `API_IMAGE=palpites-api:test`.
